1. Указатели
Определение: Указатель — это переменная, значением которой является адрес другой переменной в памяти компьютера.
Основные операции:
· Объявление: тип *имя; (например, int *p;).
· Взятие адреса (&): p = &x; (p указывает на x).
· Разыменование (*): *p = 10; (изменить значение x через указатель).
· Нулевой указатель: nullptr (C++11) или NULL.
Арифметика указателей:
· p++ — сдвиг на размер типа (sizeof(тип)).
· Связь с массивами: arr[i] эквивалентно *(arr + i).
Ссылки vs Указатели:
· Ссылка (int &ref = x;) — это псевдоним, не может быть null и не меняет адресата.
· Указатель может быть перенаправлен на другой объект.
2. Передача массива в функцию
В C/C++ массив в функцию всегда передается по указателю (адресу первого элемента). Размер массива при этом теряется.
Способы передачи:
1. Как указатель с явной передачей размера:
void func(int *arr, int size)

2. Как массив неопределенного размера (синтаксический сахар):
void func(int arr[], int size)

3. По ссылке (C++), только для массива фиксированного размера:
void func(int (&arr)[10])

Важно: При передаче многомерных массивов обязательно нужно указывать размеры всех измерений, кроме первого (самого левого), например: void func(int arr[][5], int rows).
3. Работа с файлами
Потоковые классы (C++):
· ifstream — чтение из файла (input).
· ofstream — запись в файл (output).
· fstream — чтение и запись.
Алгоритм работы:
1. Подключить .
2. Создать объект потока и открыть файл: ifstream fin("file.txt");
3. Проверить открытие: if (!fin.is_open()) // ошибка.
4. Чтение/запись (аналогично cin/cout):
· fin >> var; (до пробела).
· getline(fin, str); (строка целиком).
5. Закрытие: fin.close(); (часто автоматически в деструкторе).
Режимы открытия (флаги):
· ios::in — чтение.
· ios::out — запись (с перезаписью).
· ios::app — дозапись в конец.
· ios::binary — бинарный режим.
4. Основы ООП (Объектно-Ориентированного Программирования)
Три кита ООП:
1. Инкапсуляция: Сокрытие внутренних данных и деталей реализации. Доступ только через публичные методы. Реализуется модификаторами private, protected, public.
2. Наследование: Создание нового класса на основе существующего (повторное использование кода).
3. Полиморфизм: Способность объектов разных классов реагировать на одно и то же сообщение (вызов метода) по-разному. Реализуется через виртуальные функции (virtual).
Преимущества: Модульность, безопасность данных, упрощение поддержки кода.
5. Понятие класса
Класс — это пользовательский тип данных (шаблон, чертеж), описывающий поля (данные) и методы (функции) будущих объектов.
Структура класса:
class MyClass {
private: 
    // Данные (поля), доступные только внутри класса
    int data;
public:
    // Конструктор (вызывается при создании объекта)
    MyClass(int val) { data = val; }
    
    // Метод
    void show() { cout << data; }
    
    // Деструктор (вызывается при уничтожении)
    ~MyClass() { /* освобождение ресурсов */ }
};
Объект — это экземпляр класса (переменная типа MyClass).
MyClass obj(5);
6. Наследование в ООП
Определение: Механизм, позволяющий классу-наследнику (Derived) автоматически получить все поля и методы класса-родителя (Base), а затем дополнить или изменить их поведение.
Синтаксис (C++):
class Base {
public:
    int a;
    void func() { }
};

// public-наследование: все public члены Base становятся public в Derived
class Derived : public Base {
public:
    int b;
    void newFunc() { }
};
Виды наследования (модификаторы доступа при наследовании):
· public — права доступа из родителя не меняются.
· protected — public родителя становится protected.
· private — все члены родителя становятся private.
Ключевые моменты:
· Конструктор базового класса вызывается перед конструктором производного.
· Виртуальные функции (virtual) позволяют переопределить поведение в наследнике (override).
· Множественное наследование разрешено в C++, но требует осторожности (проблема ромба).
